Overview
What changes vs naive. Standard recurrence: T(n)=8T(n/2)+O(n²). Strassen uses 7 products instead of 8, giving T(n)=7T(n/2)+O(n²) ≈ O(n2.807).

Why stop at 2×2 leaves. Keeps arithmetic readable while showing the full structure: split → P1–P7 → recombine C11–C22.

Parallelism. The 7 products are data-independent once quadrant sums and differences are ready.
Worked Example
Load 4×4 Blocks preset. A and B split into 2×2 quadrants. Then 7 products are computed — e.g. P1=(A11+A22)(B11+B22), P4=A22·(B21−B11).

Once all 7 are ready, combine:
C11 = P1+P4−P5+P7
C12 = P3+P5
C21 = P2+P4
C22 = P1−P2+P3+P6

The Main Idea Algorithm overview
The problem. Multiplying two n×n matrices the standard way requires n³ scalar multiplications. For large matrices this becomes very expensive.

Strassen's insight (1969). When you split each matrix into four quadrant blocks, the naive approach needs 8 recursive multiplications to compute the result. Strassen found a way to get the same result using only 7 multiplications — at the cost of a few extra additions and subtractions.

Why 7 instead of 8 matters. One fewer multiplication per recursion level changes the complexity from O(n³) to O(nlog₂7) ≈ O(n2.807). This seems like a small difference, but it compounds at every level of recursion. For a 1024×1024 matrix, that is roughly 25% fewer operations.

How it works. Split A and B into four blocks each (A11, A12, A21, A22, B11, B12, B21, B22). Compute 7 specific products P1–P7 using clever sums and differences of these blocks. Then recombine P1–P7 into the four quadrants of the result: C11, C12, C21, C22. The formulas are designed so the recombination works out exactly.

Trade-off. Strassen uses fewer multiplications but more additions. For small matrices, the extra additions outweigh the savings. There is a crossover point above which Strassen becomes faster — shown in the crossover table on the left.
