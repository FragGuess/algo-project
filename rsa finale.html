Overview — Step by Step
Phase 1 — Key Generation
Step 1: Choose two distinct prime numbers p and q. In practice these are hundreds of digits long; here we use small ones to follow the math.
Step 2: Compute the public modulus n = p × q. This value is shared publicly. Its security relies on the fact that factoring n back into p and q is computationally hard for large primes.
Step 3: Compute Euler's totient φ(n) = (p−1)(q−1). This counts how many integers below n are coprime to n. It is easy to compute only if you know p and q.
Step 4: Choose a public exponent e such that 1 < e < φ(n) and gcd(e, φ(n)) = 1. This guarantees a modular inverse exists.

Phase 2 — Extended Euclidean Algorithm
Step 5: Run the Extended Euclidean Algorithm on e and φ(n). At each iteration, compute a quotient q, update the remainder r, and track Bézout coefficients s and t.
Step 6: When the remainder reaches 0, the Bézout coefficient s gives us d = e⁻¹ mod φ(n) — the private exponent. Verify: e × d ≡ 1 (mod φ(n)).
Result: Public key = (n, e), Private key = d.

Phase 3 — Encryption (Square-and-Multiply)
Step 7: To encrypt a message m, compute c = me mod n.
Step 8: Convert the exponent e to binary. Read bits from MSB to LSB. For each bit: always square the running result mod n. If the bit is 1, also multiply by the base m mod n.
Step 9: After processing all bits, the final result is the ciphertext c. This runs in O(log e) multiplications instead of e multiplications.

Phase 4 — Decryption (Square-and-Multiply)
Step 10: To decrypt, compute m = cd mod n using the same square-and-multiply method with the private exponent d.
Step 11: Process each bit of d in binary (MSB to LSB): square, then conditionally multiply by c.
Step 12: The final result recovers the original message m.

Why It Works — Correctness
Since ed ≡ 1 (mod φ(n)), we know ed = 1 + k·φ(n) for some integer k. By Euler's theorem, mφ(n) ≡ 1 (mod n) when gcd(m, n) = 1. Therefore: med = m1+k·φ(n) = m · (mφ(n))k ≡ m · 1k ≡ m (mod n). Decryption always recovers the original message.
Worked Example
Classic preset: p=17, q=11, e=7, m=88.
n = 17×11 = 187, φ(n) = 16×10 = 160.
EEA finds d = 23 (since 7×23 = 161 ≡ 1 mod 160).
Encrypt: 88⁷ mod 187 = 11.
Decrypt: 11²³ mod 187 = 88. ✓

The Main Idea Algorithm overview
The problem. Two people who have never met need to communicate securely over a public channel — without sharing a secret in advance.

RSA's insight (1977). Each person has two keys: a public key shared with everyone, and a private key kept secret. Encrypt with the public key, only the private key owner can decrypt.

The hard problem. Multiplying two large primes is easy: p × q = n. But given only n, finding p and q is computationally infeasible for large primes. This one-way difficulty is the entire security foundation.

The chain. Public key → n and e. To find private key d, you need φ(n). To find φ(n), you need p and q. To find p and q, you must factor n. If factoring n is hard, d stays secret and RSA is secure.

The pipeline. This tool walks through the full RSA process: Key Generation → Extended Euclidean Algorithm → Encryption → Decryption. Each phase is animated step by step so you can follow every calculation.
Extended Euclidean Algorithm How we find the private key d
The goal. We need to find d such that e × d ≡ 1 (mod φ(n)). In other words, d is the modular inverse of e — the number that, when multiplied by e, gives a remainder of 1 when divided by φ(n).

Why not just divide? In normal math, the inverse of 7 is 1/7. But in modular arithmetic there are no fractions — we need a whole number d. Trying every number from 1 to φ(n) would work but is far too slow for large values. The Extended Euclidean Algorithm finds d efficiently in O(log n) steps.

How it works — simple version. Start with two numbers: e and φ(n). The algorithm repeatedly divides the larger by the smaller, keeping track of remainders and two extra coefficients (s and t) at each step:

1. Divide e by φ(n) (or vice versa) — record the quotient q and remainder r.
2. Replace the larger number with the remainder. Update the coefficients s and t using the quotient.
3. Repeat until the remainder hits 0.
4. The coefficient s from the last nonzero row gives us d (adjusted to be positive mod φ(n)).

Why does this work? The algorithm maintains the identity: e × s + φ(n) × t = r at every step. When the remainder reaches 0, the previous remainder is gcd(e, φ(n)) = 1. So we get e × s + φ(n) × t = 1, which means e × s ≡ 1 (mod φ(n)). That s is our d.

Example. With e = 7 and φ(n) = 160: the algorithm runs a few division steps and finds s = 23. Check: 7 × 23 = 161 = 1 × 160 + 1, so 161 mod 160 = 1. ✓ Therefore d = 23.
Square-and-Multiply Method How we encrypt and decrypt efficiently
The problem. To encrypt, we compute me mod n. To decrypt, cd mod n. Even with small numbers like 887, the raw result is enormous (88⁷ = 40,867,559,636,992). For real RSA with 2048-bit keys, computing this the naive way (multiplying m by itself e times) is impossible.

The trick. Instead of e multiplications, we only need O(log e) — that is, roughly the number of binary digits in e. For e = 7 that is just 3 steps instead of 7. For a 2048-bit exponent, that is ~2048 steps instead of 22048.

How it works — step by step.

1. Write the exponent in binary. For example, e = 7 → 111 in binary.
2. Start with result = 1. Read the binary bits from left to right (MSB to LSB).
3. For each bit, do two things:
    — Always square: result = result² mod n
    — If the bit is 1: also multiply: result = result × base mod n
4. After processing all bits, result holds the final answer.

Why mod at every step? By taking mod n after each operation, the numbers never grow larger than n². Without this, 88⁷ would be a 14-digit number — and real RSA exponents produce numbers with millions of digits. Reducing mod n at each step keeps the numbers small and manageable.

Example. Compute 88⁷ mod 187. Exponent 7 in binary = 111.
    Bit 0 (1): square → 1, multiply → 1 × 88 = 88 mod 187 = 88
    Bit 1 (1): square → 88² = 7744 mod 187 = 77, multiply → 77 × 88 = 6776 mod 187 = 11
    Bit 2 (1): square → 11² = 121 mod 187 = 121, multiply → 121 × 88 = 10648 mod 187 = 11
    Result: 88⁷ mod 187 = 11. Done in just 3 steps.
