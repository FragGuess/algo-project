Overview

Always visible
What the algorithm does.
Bitonic sort is a comparison network. That means the wires and comparators are fixed before the input is known. For n values, the network uses O(n log^2 n) comparators and O(log^2 n) parallel depth.

Why bitonic matters.
A bitonic sequence first rises then falls, or can be rotated to have that shape. The network repeatedly builds bitonic blocks and then merges them. That is why the stage labels k and j matter: k tells you the block size, and j tells you the compare distance inside the block.

Correctness idea.
The 0-1 Principle says that if the network sorts every binary input of length n, then it also sorts every numeric input of length n.

Worked Example

Always visible
Example with 8 values.
Try the preset worst-case zigzag: 9, 1, 8, 2, 7, 3, 6, 4.

At the beginning, the network compares short pairs. Later, it compares values across larger blocks. For example, an ascending comparator between 9 and 1 keeps the smaller value 1 on the upper wire and sends 9 downward. A descending comparator does the opposite. Stage by stage, local order becomes bitonic order, then full sorted order.

Why every comparator is useful.
This zigzag pattern creates inversions at many positions, so the network cannot skip work. That is exactly why a fixed sorting network needs all of its scheduled comparators.

Reference Notes

Theory anchor

Introduced the bitonic sorting network and odd-even merge family.

CLRS, Sorting Networks.
Main ideas used here: comparison networks, bitonic merge, full bitonic sort, and the 0-1 Principle.

Complexity to mention.
Comparator count for power-of-two n: n log2(n)(log2(n)+1) / 4. Parallel depth: log2(n)(log2(n)+1) / 2.

Overview

What changes compared with the naive method.
The usual block recurrence is T(n)=8T(n/2)+O(n^2). Strassen reduces the number of recursive multiplications from 8 to 7, giving T(n)=7T(n/2)+O(n^2).

Why the page stops at 2x2 leaves.
That cutoff keeps every multiplication readable while still showing the real Strassen structure: split into quadrants, compute P1 to P7, then recombine C11, C12, C21, C22.

Parallel idea.
The seven products are independent once the needed sums and differences are prepared, so they can be computed in parallel.

Worked Example

Always visible
What to point at during the demo.
Load the 4x4 preset. First, A and B are split into four 2x2 blocks. Then the page builds P1 to P7. For example, P1 = (A11 + A22)(B11 + B22). After the seven products are ready, the result is recombined with:

C11 = P1 + P4 - P5 + P7
C12 = P3 + P5
C21 = P2 + P4
C22 = P1 - P2 + P3 + P6

This is the main calculation story the professor usually wants to see clearly.

Overview

Always visible
Key generation.
Choose two primes p and q. Compute n = pq and phi(n) = (p - 1)(q - 1). Choose e such that gcd(e, phi(n)) = 1. Then compute d = e^-1 mod phi(n) using the Extended Euclidean Algorithm.

Use of the keys.
Public key: (n, e). Private key: d. Encryption is c = m^e mod n. Decryption is m = c^d mod n.

Why it works.
RSA relies on modular arithmetic and the fact that recovering phi(n) is easy if p and q are known, but hard if n is large and unfactored.

Worked Example

Always visible
Concrete classroom example.
Take p = 17, q = 11, and e = 7.

n = 17 x 11 = 187
phi(n) = (17 - 1)(11 - 1) = 16 x 10 = 160
d = 23 because 7 x 23 = 161 and 161 mod 160 = 1

If the message is m = 88, the page computes c = 88^7 mod 187 = 11, then decrypts 11^23 mod 187 back to 88.

This gives the professor exact numbers to follow during the demo.
