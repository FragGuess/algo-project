▸ Overview — Step by Step ▾
Phase 1 — Failure Function (π table)
Step 1: Set π[0] = 0 (first char has no proper prefix-suffix).
Step 2: For i = 1 to |P|−1, pointer k tracks the current matching prefix length.
Step 3: If P[i] == P[k] → π[i] = k+1 and advance both.
Step 4: If P[i] ≠ P[k] and k > 0 → fall back: k = π[k−1].
Step 5: If P[i] ≠ P[k] and k = 0 → π[i] = 0.

Phase 2 — KMP Search
Step 6: Use i for text position, q for matched character count.
Step 7: If T[i] == P[q] → increment q.
Step 8: If q == |P| → match found at i − |P| + 1; set q = π[q−1].
Step 9: Mismatch and q > 0 → q = π[q−1]. Text pointer stays put.
Step 10: Mismatch and q = 0 → advance i.
▸ Worked Example ▾
Classic preset: T = "ABABDABACDABABCABAB", P = "ABABCABAB"
π table: [0, 0, 1, 2, 0, 1, 2, 3, 4]
π[4]=0 → "ABABC" has no proper prefix-suffix.
π[8]=4 → "ABABCABAB" has "ABAB" as its longest proper prefix-suffix.

Search: Pattern found at index 10 → T[10..18] = "ABABCABAB" ✓
KMP comparisons: ≈27  |  Naive worst-case: 18×9 = 162
Speedup: KMP is ~6× faster on this example.
▸ Naive vs KMP Comparison ▾
Naive search compares every pattern position against every text position: O(|T|·|P|). On mismatch after k chars match, it backs up the text pointer — wasting those k comparisons.

KMP never backtracks the text pointer. The π table lets KMP shift the pattern optimally: q = π[q−1] instead of restarting from 0. Text pointer i always advances.

Example: Text = "AAAAAAB", Pattern = "AAAB". Naive: 16 comparisons. KMP: 7 comparisons. The π = [0,1,2,0] lets KMP skip the redundant re-comparisons.

Worst-case savings: Pattern "AAAA" in text "AAAAAA…A": naive is O(n·m); KMP remains O(n + m).

The Main Idea Algorithm overview▾
The problem. Given text T (length n) and pattern P (length m), find all positions where P occurs in T. Naive search is O(n·m) — too slow for DNA sequences or search engines.

KMP's insight (1977). When a mismatch occurs after matching k characters, we already know those k characters. The π table encodes how much of that work is reusable: the longest suffix of what we matched that is also a prefix of the pattern. We shift to that position instead of restarting.

Two phases. Phase 1 preprocesses the pattern in O(m) time. Phase 2 scans text exactly once in O(n) time. Total: O(n + m) — linear in both input sizes.

The key invariant. At every point, q characters of the pattern match text ending at position i. When T[i] ≠ P[q], set q = π[q−1] without moving i. Safe because π[q−1] gives the longest prefix of P that is also a suffix of P[0..q−1].
Failure Function Explained How we build the π table▾
What is π[i]? The length of the longest proper prefix of P[0..i] that is also a suffix of P[0..i]. "Proper" means strictly shorter than the full string.

Example for P = "ABABCABAB":
  π[0]=0 (single char)   π[2]=1 ("ABA"→"A")   π[3]=2 ("ABAB"→"AB")   π[8]=4 ("ABABCABAB"→"ABAB")

Why this helps. When mismatch at q, T ends with P[0..q−1]. Resume at position π[q−1] in the pattern — those first π[q−1] characters are guaranteed to match.

Building π in O(m). Pointer k can only increase m times total; each fallback decreases k. So total fallbacks ≤ total increases ≤ m → O(m) overall.
KMP Search Explained How the search uses the π table▾
The scan. Walk text with pointer i; track matched chars with q. Both start at 0.

1. Compare T[i] with P[q].
2. Match → increment i and q. If q == m: full match at i−m+1. Then q = π[q−1] for overlapping matches.
3. Mismatch and q > 0 → q = π[q−1]. Don't move i. Pattern shifts right by minimum amount.
4. Mismatch and q = 0 → advance i. No prefix matches here at all.

Why we never miss a match. Every alignment skipped is provably impossible. The π table guarantees the shifted alignment is the next valid one.

Example. Text "ABABDABABC", Pattern "ABABC". After "ABAB" then 'D'≠'C': π[3]=2 → restart at q=2, not q=0. Text pointer stays put. We save 2 comparisons on every such mismatch.
