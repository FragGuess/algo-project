Overview Quick reference
Goal: Find a set C of vertices that covers every edge — every edge must have at least one endpoint in C.

Algorithm: Repeatedly pick any edge, add both endpoints to C, remove all incident edges. Repeat until no edges remain.

Guarantee: |C| ≤ 2 · OPT. Finding the exact minimum is NP-hard; this gives a provably near-optimal answer in polynomial time.
Worked Example Trace it by hand
Graph: edges {1-2, 1-3, 2-4, 3-4, 4-5, 3-5}

Step 1: Pick edge (1,2). Add vertices {1, 2} to cover. Remove all edges touching 1 or 2: removes (1-2), (1-3), (2-4). Remaining: {3-4, 4-5, 3-5}.

Step 2: Pick edge (3,4). Add vertices {3, 4} to cover. Remove all edges touching 3 or 4: removes (3-4), (4-5), (3-5). Remaining: {}.

Result: Cover = {1, 2, 3, 4}. Matching M = {(1,2),(3,4)}, so |M| = 2.
OPT ≥ |M| = 2. Cover = 4 = 2×2. Ratio = 2.

Note: The optimal cover here is {2, 3, 4} of size 3. Our algorithm gives 4 — still within the 2× bound (4 ≤ 2×3 = 6). ✓
Key Concepts Important ideas
Vertex Cover. A set C ⊆ V such that for every edge (u,v) ∈ E, u ∈ C or v ∈ C (or both). Every edge must be "covered."

Maximal Matching. A set M of edges with no two sharing an endpoint, and no edge can be added. Our algorithm implicitly builds such a matching by always picking one arbitrary uncovered edge.

Approximation Ratio. The ratio of algorithm output to optimal. APPROX-VERTEX-COVER always gives ratio ≤ 2. This is proven optimal — no polynomial algorithm with ratio < 2 is known under standard assumptions.

Why NP-hard? Minimum Vertex Cover is equivalent to Maximum Independent Set (complement): if C is a min cover, V∖C is a max independent set. Both are NP-hard to solve exactly for general graphs.

The Main Idea Algorithm overview
The problem. A vertex cover is a set of vertices such that every edge in the graph has at least one endpoint in the set. Finding the minimum vertex cover is NP-hard — no known polynomial algorithm can always find the exact optimum.

The trick. Instead of finding the exact answer, we use a greedy matching strategy that is guaranteed to be at most twice the optimal. This is the 2-approximation guarantee.

How it works. Pick any uncovered edge (u,v). Add both u and v to the cover. Remove all edges incident to u or v. Repeat until no edges remain.

Why it's a 2-approximation. Every edge we pick forms part of a matching M. No two edges in M share an endpoint, so OPT must include at least one vertex from each matched edge. That gives OPT ≥ |M|. Our cover has exactly 2|M| vertices (both endpoints of each matched edge). Therefore: |Cover| = 2|M| ≤ 2·OPT.
2-Approximation Proof Why it works
Theorem: APPROX-VERTEX-COVER returns a vertex cover of size at most 2·OPT.

Proof sketch (CLRS 35.1):

Let M be the set of edges selected by the algorithm (one per iteration). M is a matching — no two edges share a vertex, because we remove all edges incident to picked vertices immediately after picking them.

Lower bound on OPT: Any vertex cover C* must cover every edge in M. Since no two edges in M share a vertex, covering each of them requires at least one distinct vertex per edge. Therefore |C*| ≥ |M|, i.e., OPT ≥ |M|.

Upper bound on our Cover: We add exactly 2 vertices per edge in M (both endpoints). So |Cover| = 2|M|.

Combining: |Cover| = 2|M| ≤ 2·OPT. □

Also correct: The output is a valid cover because when the loop ends, every edge has been removed — meaning every edge had at least one endpoint added to C. ✓
